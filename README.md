# m-igame-archive

## 项目简介

m-igame-archive 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为 HTML 页面资料的存储与展示空间。

## 目录说明

```
├── pages/           # 存放独立 HTML 页面
├── assets/          # 页面引用的静态资源（如样式、脚本、图片等）
├── index.html       # 索引页（可选）
└── README.md        # 本文件
```

- `pages/`：主要归档目录，每个 HTML 页面可独立放置，或按子目录组织。
- `assets/`：存放页面所需的共用资源文件，避免重复引用。
- `index.html`：若需要提供页面导航或列表，可在此编写索引。

## 页面归档说明

本仓库中的 HTML 页面为独立归档文件，每个页面可包含完整的前端内容。页面之间不互相依赖，可直接通过浏览器打开使用。

- 页面文件命名建议清晰、简短，便于识别。
- 如果页面引用外部资源，请尽量使用相对路径，确保本地或静态部署时正常工作。
- 页面内容保持自包含性，避免依赖外部网络资源。

## 维护说明

- 欢迎提交 Pull Request 添加新的归档页面，或对现有页面进行修正。
- 提交前请确保页面文件结构合理，资源引用正确。
- 不建议上传包含敏感信息或违反适用法律法规的内容。
- Issue 可用于讨论页面结构、归档规范或提出改进建议。
